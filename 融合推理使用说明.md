# 融合推理 (Hybrid Inference) 使用说明

## 1. 简介

在网球/羽毛球等小目标跟踪场景中，通常会面临以下挑战：
- **全图推理精度不足**：直接在 1920x1080 原图上跑 YOLO，对于远端极小的球体容易漏检或误检。
- **局部高精度模型视野受限**：WASB 模型在裁剪图（如 586x288）上表现极佳，但无法检测画面其他区域的球。

**融合推理（Hybrid Inference）** 旨在结合两者的优势：
- **ROI 内部**（如球网附近的高频活动区域）：采用高精度的 **WASB + FP_Filter** 检测结果。
- **ROI 外部**（画面的其余部分）：采用 **YOLO** 的全图泛化检测结果。

## 2. 核心架构：离线融合 (Offline Fusion)

为了避免在同一个 Python 进程中同时加载 WASB、FP_Filter (ResNet) 和 YOLOv8 导致显存溢出（OOM）和代码强耦合，本方案采用**离线融合策略**：

1. **WASB 独立推理**：先跑完 WASB 和 FP_Filter，得到基于裁剪图坐标的 `predictions_filtered.csv`。
2. **坐标系对齐**：将 CSV 中的局部坐标映射回 1920x1080 原图，并转换为标准的 YOLO txt 格式。
3. **YOLO 动态融合**：运行 `hybrid_predict.py`，该脚本会对原图进行 YOLO 推理，**丢弃落入 ROI 区域内的 YOLO 预测框**，并读取第 2 步生成的 WASB 标签进行合并，最终输出一份完整的 YOLO 格式标签。

---

## 3. 完整工作流与操作步骤

假设我们的原图尺寸为 `1920x1080`，WASB 裁剪区域（ROI）为：`left=650, top=51, right=1236, bottom=339`。

### 第一步：获取 WASB + FP_Filter 的 CSV 结果
按照 `FP过滤二分类使用说明.md` 中的流程，完成 WASB 推理和 FP 过滤，最终得到一个过滤后的 CSV 文件，例如：
`fp_filter/patch_outputs/match1_clip1_predictions_filtered.csv`

### 第二步：将 CSV 转换为原图尺度的 YOLO 标签
使用 `csv_to_original_yolo.py` 脚本，将基于裁剪图的坐标加上偏移量（`crop-left` 和 `crop-top`），并归一化为 1920x1080 的 YOLO 格式。

```bash
python fp_filter/csv_to_original_yolo.py \
   --csv fp_filter/patch_outputs/match1_clip1_predictions_filtered.csv \
   --image-root datasets/tennis_predict \
  --output-dir fp_filter/patch_outputs/match1_clip1_orig_yolo_labels \
  --orig-w 1920 \
  --orig-h 1080 \
  --crop-left 650 \
  --crop-top 51
```
*执行后，`match1_clip1_orig_yolo_labels` 目录下会生成一系列 `.txt` 文件，这些就是 WASB 在原图尺度下的预测结果。*

### 第三步：执行融合推理
使用 `hybrid_predict.py` 脚本，结合原图、YOLO 模型和上一步生成的 WASB 标签，输出最终的融合标签。

```bash
python hybrid_predict.py \
  --input-folder datasets/tennis_predict/match1/clip1 \
  --output-folder hybrid_outputs/match1_clip1_labels \
  --yolo-model yolov8n_1280_1113.pt \
  --wasb-labels-dir fp_filter/patch_outputs/match1_clip1_orig_yolo_labels \
  --conf 0.5 \
  --orig-w 1920 --orig-h 1080 \
  --left 650 --top 51 --right 1236 --bottom 339
```

### 可选：直接使用一键 Pipeline 产出第二步结果

`run_inference_pipeline.py` 现已内置第 5 阶段，会自动调用 `csv_to_original_yolo.py`。

```bash
python run_inference_pipeline.py \
   --dataset-root datasets/tennis_predict \
   --crop-left 650 --crop-top 51 \
   --orig-w 1920 --orig-h 1080
```

执行后在时间戳输出目录下会生成：
- `stage5_original_yolo_labels/<csv_stem>_orig_yolo_labels/`

后续可将该子目录直接传给 `hybrid_predict.py` 的 `--wasb-labels-dir`。

## 4. `hybrid_predict.py` 参数详解

| 参数 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `--input-folder` | `str` | **必填** | 原始图片（1920x1080）所在的文件夹路径。 |
| `--output-folder` | `str` | **必填** | 融合后最终 YOLO txt 标签的输出目录。 |
| `--yolo-model` | `str` | **必填** | YOLOv8 模型权重路径（如 `yolov8n.pt`）。 |
| `--wasb-labels-dir` | `str` | **必填** | 第二步生成的，WASB 转换后的原图尺度 YOLO 标签目录。 |
| `--conf` | `float` | `0.5` | YOLO 推理的置信度阈值。 |
| `--orig-w` | `int` | `1920` | 原始图片的宽度。 |
| `--orig-h` | `int` | `1080` | 原始图片的高度。 |
| `--left` | `int` | `650` | ROI 区域的左边界（必须与裁剪时一致）。 |
| `--top` | `int` | `51` | ROI 区域的上边界（必须与裁剪时一致）。 |
| `--right` | `int` | `1236` | ROI 区域的右边界（必须与裁剪时一致）。 |
| `--bottom` | `int` | `339` | ROI 区域的下边界（必须与裁剪时一致）。 |
| `--inside-policy` | `str` | `center` | 判定 YOLO 框是否在 ROI 内的策略。<br>`center`: YOLO 框中心点在 ROI 内即被丢弃（推荐）。<br>`any_overlap`: YOLO 框与 ROI 有任何重叠即被丢弃。 |
| `--nms-iou` | `float` | `0.0` | 融合后执行 NMS（非极大值抑制）的 IoU 阈值。`<=0` 表示关闭 NMS。用于处理 ROI 边界处可能出现的重复框。 |
| `--no-save-empty` | `flag` | `False` | 如果指定该参数，当某帧没有任何检测结果时，不生成空的 txt 文件（默认会生成空文件以对齐帧数）。 |
| `--max-images` | `int` | `0` | 限制最多处理的图片数量。设为 `30` 可用于快速冒烟测试。`0` 表示处理全部。 |

---

## 5. 常见问题与注意事项

1. **坐标偏移量必须严格一致**
   在执行 `csv_to_original_yolo.py` 和 `hybrid_predict.py` 时，传入的 `--crop-left` / `--left` 以及 `--crop-top` / `--top` 必须与最初从视频中裁剪 ROI 时使用的坐标完全一致，否则会导致 WASB 的框在原图上发生偏移。

2. **边界重叠问题 (NMS)**
   如果一个球刚好在 ROI 的边界上，YOLO 和 WASB 可能会同时检测到它（因为 YOLO 的框中心可能刚好在 ROI 外侧，未被丢弃）。
   - **解决方案**：可以通过设置 `--nms-iou 0.45` 来开启融合后的 NMS 去重。

3. **空标签文件**
   默认情况下，即使某一帧既没有 YOLO 结果也没有 WASB 结果，脚本也会生成一个空的 `.txt` 文件。这对于后续的跟踪算法（如 ByteTrack）或评估脚本保持帧对齐非常重要。如果不需要空文件，请添加 `--no-save-empty` 参数。

4. **性能与速度**
   由于采用了离线融合，`hybrid_predict.py` 运行时只需要跑 YOLO 模型，速度非常快。WASB 的结果是直接从文本读取的，几乎不消耗额外时间。
