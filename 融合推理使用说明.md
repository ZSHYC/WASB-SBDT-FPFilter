# hybrid_predict.py 融合推理详细说明

## 一、概述

`hybrid_predict.py` 是 WASB-SBDT-FPFilter 项目的**融合推理脚本**，用于将两种检测模型（WASB 热力图检测 + YOLOv8 目标检测）的结果进行智能融合，输出最终的网球检测标注文件（YOLO 格式 `.txt`）和可视化的图片和视频。

### 核心思路

画面被分为 **ROI（感兴趣区域:Region of Interest）内** 和 **ROI 外** 两个区域，分别采用不同的融合策略：

| 区域   | 检测来源                   | 融合策略                                                      | FP 过滤方式                  |
| ------ | -------------------------- | ------------------------------------------------------------- | ---------------------------- |
| ROI 内 | WASB（优先）+ YOLO（补位） | WASB 有结果则使用 WASB；WASB 无结果时选 YOLO 置信度最高的一个 | 单目标稳定性排斥（回溯删除） |
| ROI 外 | YOLO                       | 直接使用 YOLO 全部检测框                                      | 多目标稳定性排斥（回溯删除） |

两个区域的结果合并后，经过可选的 NMS（非极大值抑制），写入最终的 YOLO 格式标注文件。

## 二、运行方式

### 基本命令

```bash
python hybrid_predict.py ^
    --input-folder <原图目录> ^
    --output-folder <输出标签目录> ^
    --yolo-model <YOLO权重路径> ^
    --wasb-labels-dir <WASB标签目录>
```

### 完整示例（推荐）

```bash
python hybrid_predict.py ^
    --input-folder datasets/tennis_predict/match1/clip1_yolo ^
    --output-folder hybrid_outputs/match1_clip1_labels ^
    --yolo-model yolov8n_1280_1113.pt ^

    --orig-w 1920 --orig-h 1080 ^

    --nms-iou 0.0 ^
```

## 三、命令行参数详解

### 3.1 必填参数

| 参数                  | 说明                                                               |
| --------------------- | ------------------------------------------------------------------ |
| `--input-folder`    | 原图文件夹路径，包含 1920×1080 的 jpg/png/bmp 图片                |
| `--output-folder`   | 融合后 YOLO 格式 `.txt` 标注文件和可视化视频图片等的输出目录     |
| `--yolo-model`      | YOLOv8 模型权重文件路径（如 `yolov8n_1280_1113.pt`）             |
| `--wasb-labels-dir` | WASB+FP_Filter 转换后的原图尺度 YOLO 标签目录（每帧一个 `.txt`） |

### 3.2 图像与 ROI 参数

| 参数         | 默认值 | 说明               |
| ------------ | ------ | ------------------ |
| `--orig-w` | 1920   | 原图宽度（像素）   |
| `--orig-h` | 1080   | 原图高度（像素）   |
| `--left`   | 650    | ROI 左边界（像素） |
| `--top`    | 51     | ROI 上边界（像素） |
| `--right`  | 1236   | ROI 右边界（像素） |
| `--bottom` | 339    | ROI 下边界（像素） |

> ROI 默认为 `(650, 51) → (1236, 339)`，即 586×288 像素区域，对应网球场的核心比赛区域。

### 3.3 YOLO 推理参数

| 参数                | 默认值     | 说明                                                                                              |
| ------------------- | ---------- | ------------------------------------------------------------------------------------------------- |
| `--conf`          | 0.5        | YOLO 置信度阈值，低于该值的检测框被丢弃                                                           |
| `--inside-policy` | `center` | 判定 YOLO 框归属 ROI 的规则：`center`（中心点在 ROI 内）或 `any_overlap`（与 ROI 有任意重叠） |
| `--nms-iou`       | 0.0        | 融合后的 NMS IoU 阈值；`<=0` 表示关闭 NMS                                                       |

### 3.4 ROI 内稳定性排斥参数

用于过滤 ROI 内 YOLO 补位结果中的固定障碍物（如帽子、球拍等静止物体）。

| 参数                             | 默认值 | 说明                                    |
| -------------------------------- | ------ | --------------------------------------- |
| `--inside-stability-window`    | 8      | 判定为静止障碍物所需的连续帧数          |
| `--inside-stability-dist`      | 15.0   | "几乎不动" 的像素距离阈值（单位：像素） |
| `--no-inside-stability-filter` | —     | 添加此参数可关闭 ROI 内稳定性排斥过滤   |

### 3.5 ROI 外稳定性排斥参数

用于过滤 ROI 外 YOLO 检测结果中的固定障碍物（如广告牌、摄像设备等）。

| 参数                               | 默认值 | 说明                                     |
| ---------------------------------- | ------ | ---------------------------------------- |
| `--outside-stability-window`     | 100    | 判定为静止障碍物所需的连续帧数           |
| `--outside-stability-dist`       | 500.0  | "几乎不动" 的像素距离阈值（单位：像素）  |
| `--outside-stability-max-misses` | 3      | 轨迹连续未匹配帧数上限，超过则删除该轨迹 |
| `--no-outside-stability-filter`  | —     | 添加此参数可关闭 ROI 外稳定性排斥过滤    |

> **为什么 ROI 外的参数比 ROI 内大很多？**
> ROI 外区域更大、可能存在的固定障碍物种类更多，且网球在 ROI 外区域移动速度通常更快，因此需要更宽松的距离阈值和更长的确认窗口来避免误删真实球。

### 3.6 可视化参数

| 参数                                             | 默认值                             | 说明                   |
| ------------------------------------------------ | ---------------------------------- | ---------------------- |
| `--visualize` / `--no-visualize`             | 开启                               | 是否生成可视化图片     |
| `--visualize-video` / `--no-visualize-video` | 开启                               | 是否生成可视化视频     |
| `--visualize-dir`                              | `<output-folder>_visualized`     | 可视化图片输出目录     |
| `--video-path`                                 | `<output-folder>_visualized.mp4` | 可视化视频输出路径     |
| `--video-fps`                                  | 25                                 | 视频帧率               |
| `--video-fourcc`                               | `mp4v`                           | 视频编码格式（4 字符） |

### 3.7 其他参数

| 参数                | 默认值 | 说明                                             |
| ------------------- | ------ | ------------------------------------------------ |
| `--no-save-empty` | 否     | 添加此参数后，无检测结果的帧不写空 `.txt` 文件 |
| `--max-images`    | 0      | 最多处理多少张图片；`0` 表示全部处理           |

## 四、三阶段执行流程

脚本采用**三阶段架构**执行，核心是「先推理打标签，再回溯删除，最后批量写入」：

```
┌─────────────────────────────────────────────────────────────────┐
│  Phase 1: 推理 + 打标签                                         │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  逐帧处理：                                                │  │
│  │  1. YOLO 全图推理 → 像素坐标检测框                          │  │
│  │  2. 按 ROI 分流为 inside / outside                         │  │
│  │  3. ROI 外：打 track_id 标签（不过滤）                      │  │
│  │  4. ROI 内：WASB 有结果→标记 None；无结果→YOLO 补位打       │  │
│  │     cluster_id 标签（不过滤）                               │  │
│  │  5. 缓存 (stem, tagged_outside, tagged_roi) 到内存          │  │
│  └───────────────────────────────────────────────────────────┘  │
│                            ↓                                     │
│  Phase 2: 回溯删除 + 批量写入                                    │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  1. 获取 confirmed_static_ids（ROI 内 + ROI 外）           │  │
│  │  2. 遍历所有缓存帧：                                       │  │
│  │     - 删除 track_id ∈ outside_static_ids 的 ROI 外检测框    │  │
│  │     - 删除 cluster_id ∈ inside_static_ids 的 ROI 内补位     │  │
│  │     - WASB 结果（标签=None）永远保留                        │  │
│  │  3. 合并 + NMS → 写入 YOLO format .txt                     │  │
│  └───────────────────────────────────────────────────────────┘  │
│                            ↓                                     │
│  Phase 3: 可视化                                                 │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  读取已写入的 .txt 标签 → 在原图上绘制检测框                 │  │
│  │  → 输出可视化图片 + 视频                                    │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

## 五、融合逻辑详解

### 5.1 ROI 内融合

```
某帧 ROI 内有 WASB 结果？
    ├── 是 → 使用 WASB 全部结果，丢弃该帧 ROI 内所有 YOLO 框
    │         标签 = None（永远不会被回溯删除）
    │
    └── 否 → ROI 内有 YOLO 框？
              ├── 是 → 取置信度最高的 1 个 YOLO 框作为补位
              │         打上 cluster_id 标签（Phase 2 判断是否为障碍物）
              │         其余 YOLO 框丢弃
              │
              └── 否 → 该帧 ROI 内无检测结果
```

**设计理由**：

- WASB+FP_Filter 在 ROI 内的小球检测精度通常优于 YOLO（基于高分辨率热力图）
- YOLO 仅作为 WASB 漏检时的补位方案
- 补位只取最高置信度的一个，避免多个 YOLO 框引入噪声

### 5.2 ROI 外处理

```
某帧 ROI 外的所有 YOLO 框
    → 全部保留并打上 track_id 标签
    → Phase 2 回溯删除已确认的静止轨迹
```

ROI 外不使用 WASB（WASB 只检测 ROI 内区域），完全依赖 YOLO。

## 六、回溯删除机制

### 6.1 问题背景

在线过滤（逐帧判断）的固有缺陷：当一个固定障碍物首次出现时，过滤器需要积累 `window` 帧的历史才能确认它是静止的。这意味着前 `window - 1` 帧中该障碍物不会被过滤，造成**误检泄漏**。

**示例**：`window=100` 时，一个始终存在的帽子会在前 99 帧中被保留为检测结果。

### 6.2 回溯删除方案

回溯删除彻底解决了这个问题：

1. **Phase 1（推理阶段）**：不做任何过滤，只为每个检测框打上标识标签（track_id / cluster_id），同时在内存中缓存所有帧的结果。
2. **Phase 2（回溯阶段）**：处理完所有帧后，获取已确认的静止 ID 集合，回到第 1 帧起遍历所有缓存结果，删除属于这些 ID 的全部检测框。

**效果**：即使 `window=100`，一旦第 100 帧确认了某位置是障碍物，前 99 帧中该位置的检测框也会被彻底清除。

### 6.3 ROI 内 — StabilityFilter（单目标，位置聚类）

**适用场景**：ROI 内每帧最多一个 YOLO 补位候选。

**工作流程**：

1. 维护一个滑动窗口历史 `deque(maxlen=window)`，存储最近 `window` 帧的补位像素坐标
2. 每次补位时，如果当前位置与历史中所有点距离均 > `dist`（全新位置出现），清空历史并分配新聚类 ID
3. 如果当前位置与历史中所有点距离均 ≤ `dist` 且窗口已满，则确认当前聚类为静止障碍物
4. 返回当前聚类 ID 用于标签

**关键设计**：

- 出现新位置时会**清空历史重新计数**，防止旧障碍物的记录干扰后续出现的真实球
- WASB 结果的标签始终为 `None`，永远不会被回溯删除

### 6.4 ROI 外 — MultiObjectStabilityFilter（多目标，轨迹追踪）

**适用场景**：ROI 外可能同时存在多个检测框。

**工作流程**：

1. 维护多条轨迹 `tracks`，每条轨迹有唯一 ID、中心坐标、命中计数 `hits`、缺失计数 `misses`
2. 每帧将所有 ROI 外的 YOLO 检测框与现有轨迹进行**贪心距离匹配**：
   - 计算所有 (检测框, 轨迹) 对的像素距离
   - 按距离从小到大排序，贪心分配（每个检测框和轨迹最多匹配一次）
3. 匹配成功：更新轨迹位置，`hits += 1`，`misses = 0`
4. 未匹配的检测框：创建新轨迹（`hits=1`）
5. 未匹配的轨迹：`misses += 1`，超过 `max_misses` 则删除
6. `hits >= window` 时，该轨迹 ID 加入 `confirmed_static_ids`

## 九、调参建议

### 9.1 ROI 内参数

| 场景                   | 建议                                     |
| ---------------------- | ---------------------------------------- |
| 障碍物较少、补位较精准 | `window=8`, `dist=15`（默认值即可）  |
| 补位结果中障碍物较多   | 减小 `window`（如 5），加快确认速度    |
| 球在 ROI 内移动缓慢    | 减小 `dist`（如 10），更严格判定"不动" |
| 不希望过滤任何补位     | 添加 `--no-inside-stability-filter`    |

### 9.2 ROI 外参数

| 场景                     | 建议                                                       |
| ------------------------ | ---------------------------------------------------------- |
| 固定障碍物多（广告牌等） | `window=100`, `dist=500`（默认值）                     |
| 视频较短（几百帧）       | 适当减小 `window`（如 50），否则可能无法在视频结束前确认 |
| 误删快速移动的球         | 减小 `dist`，缩小"静止"判定范围                          |
| 不希望过滤任何 ROI 外框  | 添加 `--no-outside-stability-filter`                     |

### 9.3 注意事项

- 回溯删除是**全局性**的：一旦确认某个位置是障碍物，该位置在所有帧中的检测框都会被删除
- `window` 值越大，误删风险越低，但需要更多帧才能开始过滤——不过因为有回溯删除，一旦确认就从第 1 帧起彻底清除
- 建议先用默认参数运行，观察可视化视频后再调整
- `--max-images` 参数可用于快速验证，处理前 N 帧后即可查看效果
